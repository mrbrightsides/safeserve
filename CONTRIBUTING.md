First off, thank you for considering contributing to **SafeServe MBG**! Our mission is to ensure that Indonesia's children receive safe and nutritious meals through technology.

## How to Contribute

### 1. Reporting Bugs
*   Check the [Issues](https://github.com/mrbrightsides/safeserve/issues) to see if the bug has already been reported.
*   If not, open a new issue. Include a clear title, description, and steps to reproduce the error.

### 2. Suggesting Enhancements
*   We welcome new ideas for safety features (e.g., new AI vision checks, local language support).
*   Open an issue with the "enhancement" label to discuss your idea.

### 3. Pull Requests
1.  Fork the repo and create your branch from `main`.
2.  If you've added code that should be tested, add tests!
3.  Ensure your code follows the existing style (we use Prettier and ESLint).
4.  Write a clear, concise PR description.

## Coding Standards
*   **TypeScript**: All new code must be type-safe. Avoid `any`.
*   **Tailwind CSS 4**: Use utility classes for styling. Follow the "Command Center" aesthetic.
*   **AI Safety**: When adding new AI functions, always include a **fallback** mechanism in case of API failures/quota limits.
